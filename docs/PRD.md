# Product Requirements Document (PRD)

## 1. Overview

Goalnity는 아마추어 축구 동호인을 위한 **팀 매칭 및 구장 현황 안내 통합 플랫폼**이다. 팀을 등록하고, 지역 공식 체육시설 예약 시스템과 연동해 구장 현황을 한눈에 파악하며, 상대 팀과 매칭해 경기를 진행한 뒤 결과를 쌓아가는 커뮤니티형 서비스다.

**MVP는 노원구를 시작점으로** 서비스를 검증하고, 이후 서울 전 자치구 → 수도권 → 전국으로 단계적 확장을 목표로 한다. 구장 예약은 각 지역 공식 예약 채널을 그대로 활용하고, Goalnity는 **대관 현황 조회 + 예약 페이지 바로가기 + 팀 매칭 + 기록 축적**에 집중해 아마추어 축구 생태계를 활성화하는 것이 핵심 가치다.

## 2. Problem Statement

### 현재 문제점

- 노원구 내 풋살장·축구장 대관 정보가 각 시설 홈페이지와 노원구 예약 시스템에 분산되어 있어 **어디서 예약해야 하는지 파악 불가**
- 아마추어 팀 간 경기 상대를 구하는 공식 채널이 없어 **네이버 카페(모두의 풋살,축구 등), 다음 카페(서경카페 등) 등 분산된 커뮤니티에 의존**
- 상대 팀의 실제 실력을 사전에 검증할 방법이 없어 **선수 출신 은폐, 팀 레벨 허위 등록 등 실력 속임 문제가 빈번**해 경기 후 분쟁 발생
- 팀별 전적, 승률, 선수 기록이 체계적으로 관리되지 않아 **데이터 축적이 어려움**
- 팀 소속 선수 등록·관리가 구두/메신저로 이뤄져 **명단 관리에 혼선 발생**
- 동호인 간 정보 공유 커뮤니티가 파편화되어 있어 **선수 영입, 용병 구인, 장비 거래 등이 불편**

### 핵심 Pain Point

| 대상 | Pain Point |
|------|------------|
| 팀 매니저 | 노원구 예약 사이트에서 어떤 구장이 비어있는지 한눈에 파악하기 어려움 |
| 팀 매니저 | 상대 팀을 네이버/다음 카페에서 직접 찾아야 하며, 매칭 후 실력 속임(선수 출신 은폐 등)으로 인한 분쟁이 잦음 |
| 선수 | 내 팀의 전적·개인 기록을 확인할 방법 없음 |
| 신규 팀 | 팀 등록 및 리그 참여 절차가 불투명함 |

## 3. Product Goals

1. **대관 현황 안내** — 노원구 공식 예약 시스템의 구장별 예약 현황을 한 곳에서 확인하고, 예약 페이지로 바로 연결
2. **팀 생태계 구축** — 아마추어 팀의 등록, 선수 관리, 이력 관리를 체계화
3. **신뢰 기반 매칭** — 실력·지역·일정 기반 매칭 + 경기 이력 데이터로 실력 허위 등록을 방지하고 공정한 상대를 연결
4. **데이터 축적** — 매치 결과를 꾸준히 쌓아 팀·선수 통계 데이터베이스 구축
5. **커뮤니티 활성화** — 용병 구인, 정보 공유, 선수 영입 등 동호인 커뮤니티 제공
6. **관리 부담 최소화** — 팀 매니저의 수동 입력을 최대한 줄이고, 반복 작업은 자동화해 지속 사용을 유도

## 4. Target Users & Personas

### Primary: 노원구 아마추어 축구·풋살 동호인

#### Persona 1 — 팀 매니저 "현수" (32세)

- 5년 된 직장인 풋살팀 운영 중 (12명)
- 매주 수요일 저녁 정기 구장을 잡는 게 큰 스트레스
- 친선 경기 상대를 구할 때 네이버 카페·다음 카페를 뒤지다 지침
- 지난번에 만난 팀이 선수 출신을 3명이나 숨겼다가 경기 후 크게 싸운 적 있음
- **니즈**: 노원구 구장 예약 가능 시간대를 앱에서 빠르게 파악하고, 실제 전적과 레벨이 검증된 상대 팀과 매칭되길 원함

#### Persona 2 — 열성 선수 "지원" (27세)

- 주말마다 2~3팀에서 용병으로 뛰는 풋살 마니아
- 자신의 경기 기록(득점, 어시스트, 출전 횟수)을 정리하고 싶음
- **니즈**: 내 플레이 통계를 확인하고, 용병 요청이 들어오면 알림을 받고 싶음

#### Persona 3 — 신규 팀 창단자 "민아" (30세)

- 회사 동료들과 첫 팀을 만들려 함
- 어디서 어떻게 팀을 등록하고 경기를 잡는지 전혀 모름
- **니즈**: 팀 등록부터 첫 경기까지 가이드가 필요

## 5. User Journeys

### Journey 1: 구장 현황 확인 및 예약 (Core)

```
메인 페이지 접속
    → 날짜/시간/유형 필터 선택
    → 노원구 구장 목록 + 예약 가능 현황 카드 확인
    → 원하는 구장 선택
    → [노원구 예약 바로가기] 버튼 클릭
    → 노원구 공식 예약 페이지로 이동해 예약 완료
```

### Journey 2: 선수 프로필 자동 생성

```
매치 확정
    → 시스템이 경기 초대 링크 자동 생성
    → 매니저가 단체 카톡방에 링크 공유
    → 선수들이 링크 탭 → 앱 가입/로그인 → 경기 참여자로 자동 등록
    → 경기 결과 확정
    → 참여 등록된 선수들 출전 횟수 자동 반영
    → (선택) 득점자 입력 시 개인 득점 통계 누적
```

### Journey 3: 상대 팀 매칭

```
매칭 탭 접속
    → 팀 레벨 / 희망 날짜·시간 / 구장 입력
    → 시스템이 조건에 맞는 팀 목록 추천
    → 매칭 요청 발송
    → 상대 팀 수락 → 매치 확정 알림
    → 경기 당일 결과 입력
```

### Journey 3: 경기 결과 등록

```
완료된 매치 탭 접속
    → 경기 결과 입력 (스코어, 득점자, MOM)
    → 상대 팀 매니저 결과 확인 · 승인
    → 팀·선수 통계 자동 업데이트
    → 리더보드 순위 반영
```

### Journey 4: 커뮤니티 활동

```
커뮤니티 탭 접속
    → 카테고리 선택 (용병 구인, 팀원 모집, 자유 게시판, 장비 거래)
    → 글 작성 / 댓글 / DM 기능으로 소통
```

## 7. Functional Requirements

### FR-01: 노원구 대관 현황

- 노원구 공설 체육시설(풋살장·축구장) 목록 표시
- 각 구장별 날짜/시간대별 예약 가능 여부 표시 (초록: 가능, 빨강: 마감, 회색: 미운영)
  - 현황 데이터는 노원구 공식 예약 시스템 페이지를 참조해 주기적으로 동기화
- 필터: 날짜, 시간대, 구장 유형(풋살/축구), 실내/실외
- 구장 카드: 구장명, 주소, 면 수, 조명 여부, 예약 현황 요약
- 구장 상세 페이지에서 **[노원구 공식 예약 페이지로 이동]** 버튼 제공 → 노원구 체육시설 예약 시스템으로 외부 링크
- Goalnity 내 자체 예약·결제 기능 없음 (예약은 노원구 공식 채널에서 진행)

### FR-02: 팀 등록 관리

**최소 필수 입력 원칙** — 처음부터 모든 정보를 요구하지 않고, 단계적으로 채워나갈 수 있도록 설계

- 팀 생성 시 필수 입력: **팀명, 레벨(입문/초급/중급/상급)** 2가지만
  - 로고, 활동 지역, 연락처 등은 선택 사항으로 나중에 추가 가능
- 선수 등록: **이름만 필수**, 나머지(포지션·등번호·연락처)는 선택
  - 선수 없이도 팀 생성 및 매칭 가능 (인원만 입력)
- 팀 매니저 권한은 경기 결과 승인 1가지로 단순화

**선수 프로필 생성 방식 — 경기 초대 링크**
1. 매치 확정 시 → 시스템이 해당 경기 전용 초대 링크 자동 생성
2. 매니저가 기존 단체 카톡방에 링크 공유 (추가 작업 없음)
3. 선수가 링크 탭 → 앱 가입 또는 로그인 → 해당 경기 참여자로 자동 등록
4. 경기 결과 확정 시 → 참여 등록된 선수들의 출전 횟수 자동 +1
5. 결과 입력 시 득점자·어시스트 선택하면 → 해당 선수 개인 통계 자동 누적
- 앱 미가입 선수: 팀 통계(승패)에만 반영, 개인 통계 없음 (가입 시 소급 불가)
- 매니저가 선수 명단을 별도로 앱에 등록할 필요 없음

### FR-03: 매칭 시스템

**3탭 매칭 원칙** — 매칭 요청까지 최대 3번의 탭으로 완료

- 매칭 요청 필수 입력: **날짜, 경기 방식(5vs5/7vs7/11vs11)** 2가지만
  - 장소·시간은 선택 사항, 미입력 시 "노원구 전체 / 시간 협의" 로 자동 설정
- **매칭 프리셋**: 이전 매칭 조건을 저장해 다음엔 한 번에 재사용
- 팀 레벨 필터: ±1 레벨 내 팀만 추천 (기본값, 변경 가능)
- **레벨 신뢰도 시스템**: 누적 경기 결과 기반으로 팀 레벨 자동 조정 — 초기 자기 신고 레벨은 경기 수 5회 미만 팀에 "(미검증)" 배지 표시
- **선수 이력 신고 제도**: 매칭 후 상대 팀의 실력 속임이 의심될 경우 신고 가능. 누적 신고 시 레벨 재검토 및 경고
- 매칭 요청 시 상대 팀의 최근 5경기 전적·득실·레벨 변동 이력 공개
- 매칭 요청 상태: 대기 중 / 수락 / 거절 / 만료 (24시간 내 응답 없으면 자동 만료)
- 매칭 확정 후 양팀에게 일정 알림 발송

### FR-04: 경기 결과 및 데이터 축적

**결과 입력 최소화 원칙** — 스코어만 입력해도 기록이 쌓이도록 설계

- 경기 결과 입력: **스코어만 필수** (득점자·어시스트·MOM은 선택, 나중에 추가 가능)
- **단방향 확인 방식**: 한 팀이 스코어를 입력하면 상대 팀은 "맞음 / 다름" 버튼 하나만 누르면 확정
  - 상대 팀이 6시간 내 응답 없으면 자동 확정 (분쟁 없을 때 매니저 추가 액션 불필요)
  - "다름" 선택 시에만 양팀 협의 플로우 진행
- 결과 미입력 시 경기 종료 2시간 후 알림 1회 발송 (반복 알림 없음)
- 확정된 경기 데이터: 팀 통계(승·무·패, 득실점, 승률) 및 선수 통계 자동 반영
- 리더보드: 팀 승점 순, 득점왕, 어시스트 순위

### FR-05: 커뮤니티

- 카테고리: 용병 구인 / 팀원 모집 / 자유게시판 / 장비 거래
- 게시글: 제목, 내용, 이미지 첨부 (최대 5장), 태그
- 용병 구인 글에는 희망 날짜·포지션·연락처 필드 추가
- DM(1:1 채팅) 기능
- 신고 및 차단 기능

## 8. Non-Functional Requirements

| 항목 | 요구사항 |
|------|---------|
| **성능** | 구장 현황 페이지 LCP 2초 이내, 매칭 검색 응답 1초 이내 |
| **현황 갱신** | 구장 예약 현황 노원구 공식 페이지 기준으로 주기적 갱신 (최소 1시간 1회), 갱신 시각 표시 |
| **확장성** | 구장 수·팀 수 10배 증가에도 성능 저하 없는 구조 |
| **보안** | 개인정보(연락처) 마스킹 처리, JWT 인증 |
| **가용성** | 99.5% uptime, 주말 피크타임(금·토 18~22시) 안정적 처리 |
| **반응형** | 모바일(360px~), 태블릿(768px~), 데스크톱(1024px~) |
| **알림** | 매칭 요청·확정·결과 입력 요청 푸시 알림 5분 이내 발송 |

  stats: TeamStats;
  createdAt: string;
}

  id: string;
  userId?: string;                     // 앱 회원이면 연동

  name: string;
  number?: number;                     // 등번호

interface Match {
  id: string;
  homeTeamId: string;
  awayTeamId: string;
  venueId: string;
  surfaceId: string;
  scheduledAt: string;                 // ISO 8601
  format: MatchFormat;                 // 경기 방식
  status: MatchStatus;
  result?: MatchResult;
}

type MatchFormat = "5v5" | "7v7" | "11v11";
type MatchStatus = "pending" | "confirmed" | "completed" | "cancelled";

interface MatchResult {
  homeScore: number;
  awayScore: number;
  scorers: Scorer[];
  assists: Assist[];
  mom?: string;                        // Player id

  confirmedByAway: boolean;

}

  wins: number;

  losses: number;
  goalsFor: number;
  goalsAgainst: number;
  points: number;                      // 승점 (승3, 무1, 패0)
}

interface PlayerStats {
  appearances: number;
  goals: number;
  assists: number;
  momCount: number;
}

## 12. UI/UX Principles

### 디자인 원칙

1. **즉시 파악** — 구장 예약 가능 여부를 색상으로 즉시 구분 (초록/빨강)
2. **모바일 우선** — 이동 중에도 구장 검색·매칭 요청이 가능한 터치 친화적 UI
3. **최소 마찰** — 매칭 요청까지 3탭 이내, 결과 입력은 스코어 하나로 완료
4. **자동화 우선** — 반복 입력(매칭 프리셋), 자동 확정(6시간 미응답), 통계 자동 집계로 매니저 개입 최소화
5. **점진적 정보 수집** — 팀 생성·매칭·결과 입력 시 필수 항목만 요구하고, 나머지는 언제든 나중에 채울 수 있도록
6. **투명한 데이터** — 팀·선수 통계는 항상 최신 경기 반영 여부를 표시
7. **커뮤니티 신뢰** — 실명 팀 기반 활동으로 신뢰성 확보

## 14. Risks & Assumptions

### Assumptions (가정)

- 노원구 공설 구장 예약 현황은 노원구 공식 예약 시스템 페이지에서 주기적으로 수집 가능
- 예약 행위 자체는 노원구 공식 채널에서 이뤄지므로 자체 결제·예약 시스템 불필요
- 아마추어 팀 매니저들이 앱을 통한 매칭 방식에 적응 의지 있음

### Risks (위험 요소)

| 위험 | 영향 | 대응 |
|------|------|------|
| **구장 현황 동기화** — 노원구 공식 페이지 구조 변경 시 수집 로직 깨짐 | 중간 | 정기 모니터링, 수동 등록 폴백 |
| **콜드 스타트** — 초기 팀 수 부족으로 매칭 어려움 | 높음 | 네이버·다음 카페 커뮤니티 홍보, 초기 팀 인센티브 제공 |
| **실력 속임** — 선수 출신 은폐, 팀 레벨 허위 등록으로 공정 매칭 저해 | 높음 | 경기 결과 누적 기반 자동 레벨 조정, 신고 제도, 미검증 팀 배지 표시 |
| **결과 미입력** — 경기 후 결과 등록 이탈 | 중간 | 자동 리마인더 알림, 입력 시 포인트 지급 |
| **정보 신뢰성** — 취소 없이 노쇼 발생 | 중간 | 노쇼 패널티 제도, 평판 점수 도입 |
| **현황 정보 오차** — 동기화 주기 내 실제 예약 상태와 불일치 | 낮음 | 갱신 시각 명시, "노원구 공식 페이지에서 최종 확인" 안내 문구 표시 |
