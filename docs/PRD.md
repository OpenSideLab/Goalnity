# Product Requirements Document (PRD)

## 1. Overview

Goalnity는 노원구 아마추어 축구 동호인을 위한 **팀 매칭 및 풋살/축구장 예약 통합 플랫폼**이다. 팀을 등록하고, 노원구 내 대관 가능한 구장 현황을 실시간으로 확인하며, 상대 팀과 매칭해 경기를 진행한 뒤 결과를 쌓아가는 커뮤니티형 서비스다.

흩어진 대관 정보, 상대 팀 구하기의 어려움, 경기 기록 부재를 **단일 플랫폼으로 해결**해 노원구 아마추어 축구 생태계를 활성화하는 것이 핵심 가치다.

---

## 2. Problem Statement

### 현재 문제점

- 노원구 내 풋살장·축구장 대관 정보가 각 시설 홈페이지에 분산되어 있어 **한눈에 파악 불가**
- 아마추어 팀 간 경기 상대를 구하는 공식 채널이 없어 **카카오톡 오픈채팅 등 비공식 수단에 의존**
- 팀별 전적, 승률, 선수 기록이 체계적으로 관리되지 않아 **데이터 축적이 어려움**
- 팀 소속 선수 등록·관리가 구두/메신저로 이뤄져 **명단 관리에 혼선 발생**
- 동호인 간 정보 공유 커뮤니티가 파편화되어 있어 **선수 영입, 용병 구인, 장비 거래 등이 불편**

### 핵심 Pain Point

| 대상 | Pain Point |
|------|------------|
| 팀 매니저 | 구장 예약 가능 여부를 일일이 전화로 확인해야 함 |
| 팀 매니저 | 실력이 비슷한 상대 팀을 찾는 공식 채널 없음 |
| 선수 | 내 팀의 전적·개인 기록을 확인할 방법 없음 |
| 신규 팀 | 팀 등록 및 리그 참여 절차가 불투명함 |
| 구장 담당자 | 예약·취소 현황을 효율적으로 관리하기 어려움 |

## 3. Product Goals

1. **대관 현황 통합** — 노원구 공·사설 구장의 실시간 예약 가능 현황을 한 곳에서 확인
2. **팀 생태계 구축** — 아마추어 팀의 등록, 선수 관리, 이력 관리를 체계화
3. **스마트 매칭** — 실력·지역·일정 기반으로 최적의 상대 팀을 자동 추천
4. **데이터 축적** — 매치 결과를 꾸준히 쌓아 팀·선수 통계 데이터베이스 구축
5. **커뮤니티 활성화** — 용병 구인, 정보 공유, 선수 영입 등 동호인 커뮤니티 제공

## 4. Target Users & Personas

### Primary: 노원구 아마추어 축구·풋살 동호인

#### Persona 1 — 팀 매니저 "현수" (32세)

- 5년 된 직장인 풋살팀 운영 중 (12명)
- 매주 수요일 저녁 정기 구장을 잡는 게 큰 스트레스
- 친선 경기 상대를 구할 때 오픈채팅방을 뒤지다 지침
- **니즈**: 구장 예약 가능 시간대를 앱에서 바로 확인하고, 비슷한 수준의 상대 팀과 매칭되길 원함

#### Persona 2 — 열성 선수 "지원" (27세)

- 주말마다 2~3팀에서 용병으로 뛰는 풋살 마니아
- 자신의 경기 기록(득점, 어시스트, 출전 횟수)을 정리하고 싶음
- **니즈**: 내 플레이 통계를 확인하고, 용병 요청이 들어오면 알림을 받고 싶음

#### Persona 3 — 구장 운영자 "상철" (45세)

- 노원구 실내 풋살장 운영 (4면)
- 전화·문자로만 예약 받다 보니 누락·중복 예약이 자주 발생
- **니즈**: 온라인 예약 시스템과 실시간 현황판으로 운영 효율화

#### Persona 4 — 신규 팀 창단자 "민아" (30세)

- 회사 동료들과 첫 팀을 만들려 함
- 어디서 어떻게 팀을 등록하고 경기를 잡는지 전혀 모름
- **니즈**: 팀 등록부터 첫 경기까지 가이드가 필요

## 5. User Journeys

### Journey 1: 구장 예약 (Core)

```
메인 페이지 접속
    → 날짜/시간/지역 필터 선택
    → 노원구 구장 목록 + 예약 가능 슬롯 확인
    → 원하는 구장 · 시간 선택
    → 팀 로그인 후 예약 확정
    → 예약 완료 알림 수신
```

### Journey 2: 상대 팀 매칭

```
매칭 탭 접속
    → 팀 레벨 / 희망 날짜·시간 / 구장 입력
    → 시스템이 조건에 맞는 팀 목록 추천
    → 매칭 요청 발송
    → 상대 팀 수락 → 매치 확정 알림
    → 경기 당일 결과 입력
```

### Journey 3: 경기 결과 등록

```
완료된 매치 탭 접속
    → 경기 결과 입력 (스코어, 득점자, MOM)
    → 상대 팀 매니저 결과 확인 · 승인
    → 팀·선수 통계 자동 업데이트
    → 리더보드 순위 반영
```

### Journey 4: 커뮤니티 활동

```
커뮤니티 탭 접속
    → 카테고리 선택 (용병 구인, 팀원 모집, 자유 게시판, 장비 거래)
    → 글 작성 / 댓글 / DM 기능으로 소통
```

## 7. Functional Requirements

### FR-01: 노원구 대관 현황

- 노원구 내 등록된 구장(공설·사설) 목록 표시
- 각 구장별 날짜/시간대별 예약 가능 여부 표시 (초록: 가능, 빨강: 불가, 회색: 미운영)
- 필터: 날짜, 시간대, 구장 유형(풋살/축구), 실내/실외, 면 수
- 구장 상세: 주소, 주차, 면 크기, 조명 여부, 가격, 연락처, 사진

### FR-02: 팀 등록 관리

- 팀 생성 시 필수 입력: 팀명, 로고, 대표 연락처, 활동 지역, 평균 나이대, 레벨(입문/초급/중급/상급)
- 선수 등록: 이름, 포지션, 등번호, 연락처 (선택)
- 팀 매니저 권한: 선수 추가/삭제, 경기 결과 최종 승인
- 선수 개인 프로필: 소속 팀, 포지션, 출전 경기 수, 득점·어시스트 누계

### FR-03: 매칭 시스템

- 매칭 요청 입력 항목: 희망 날짜·시간, 장소(구장 또는 구단위), 인원 수, 경기 방식(풋살 5vs5 / 7vs7 / 11vs11)
- 팀 레벨 필터: ±1 레벨 내 팀만 추천 (기본값, 변경 가능)
- 매칭 요청 상태: 대기 중 / 수락 / 거절 / 만료 (24시간 내 응답 없으면 자동 만료)
- 매칭 확정 후 양팀에게 일정 알림 발송

### FR-04: 경기 결과 및 데이터 축적

- 경기 결과 입력: 최종 스코어, 득점자, 어시스트, MOM(최우수선수)
- 양팀 매니저 모두 승인해야 공식 기록으로 확정
- 결과 미입력 시 경기 종료 2시간 후 자동 알림
- 확정된 경기 데이터: 팀 통계(승·무·패, 득실점, 승률) 및 선수 통계 자동 반영
- 리더보드: 팀 승점 순, 득점왕, 어시스트 순위

### FR-05: 커뮤니티

- 카테고리: 용병 구인 / 팀원 모집 / 자유게시판 / 장비 거래
- 게시글: 제목, 내용, 이미지 첨부 (최대 5장), 태그
- 용병 구인 글에는 희망 날짜·포지션·연락처 필드 추가
- DM(1:1 채팅) 기능
- 신고 및 차단 기능

## 8. Non-Functional Requirements

| 항목 | 요구사항 |
|------|---------|
| **성능** | 구장 현황 페이지 LCP 2초 이내, 매칭 검색 응답 1초 이내 |
| **실시간성** | 구장 예약 현황 5분 이내 갱신 |
| **확장성** | 구장 수·팀 수 10배 증가에도 성능 저하 없는 구조 |
| **보안** | 개인정보(연락처) 마스킹 처리, JWT 인증 |
| **가용성** | 99.5% uptime, 주말 피크타임(금·토 18~22시) 안정적 처리 |
| **반응형** | 모바일(360px~), 태블릿(768px~), 데스크톱(1024px~) |
| **알림** | 매칭 요청·확정·결과 입력 요청 푸시 알림 5분 이내 발송 |

## 9. Data Model

### 핵심 엔티티

```typescript
// 구장
interface Venue {
  id: string;
  name: string;                        // 구장명
  address: string;                     // 주소 (노원구 내)
  type: "futsal" | "soccer";          // 풋살 / 축구
  indoor: boolean;                     // 실내 여부
  surfaces: Surface[];                 // 각 면 정보
  phone: string;
  pricePerHour: number;               // 시간당 가격 (원)
  photos: string[];
  operatingHours: OperatingHour[];
}

interface Surface {
  id: string;
  venueId: string;
  name: string;                        // 예: "A구장", "1번 코트"
  capacity: number;                    // 최대 인원
  width: number;                       // 가로 (m)
  length: number;                      // 세로 (m)
  hasLighting: boolean;
}

interface TimeSlot {
  surfaceId: string;
  date: string;                        // ISO 8601
  startTime: string;                   // "HH:mm"
  endTime: string;
  status: "available" | "reserved" | "closed";
  reservationId?: string;
}

// 팀
interface Team {
  id: string;
  name: string;
  logoUrl?: string;
  level: TeamLevel;                    // 레벨
  region: string;                      // 주 활동 지역
  avgAge?: number;
  managerId: string;                   // 팀 매니저 userId
  players: Player[];
  stats: TeamStats;
  createdAt: string;
}

type TeamLevel = "beginner" | "amateur" | "intermediate" | "advanced";

interface Player {
  id: string;
  userId?: string;                     // 앱 회원이면 연동
  teamId: string;
  name: string;
  number?: number;                     // 등번호
  position?: Position;
  stats: PlayerStats;
}

type Position = "GK" | "DF" | "MF" | "FW";

// 매치
interface Match {
  id: string;
  homeTeamId: string;
  awayTeamId: string;
  venueId: string;
  surfaceId: string;
  scheduledAt: string;                 // ISO 8601
  format: MatchFormat;                 // 경기 방식
  status: MatchStatus;
  result?: MatchResult;
}

type MatchFormat = "5v5" | "7v7" | "11v11";
type MatchStatus = "pending" | "confirmed" | "completed" | "cancelled";

interface MatchResult {
  homeScore: number;
  awayScore: number;
  scorers: Scorer[];
  assists: Assist[];
  mom?: string;                        // Player id
  confirmedByHome: boolean;
  confirmedByAway: boolean;
  confirmedAt?: string;
}

// 통계
interface TeamStats {
  wins: number;
  draws: number;
  losses: number;
  goalsFor: number;
  goalsAgainst: number;
  points: number;                      // 승점 (승3, 무1, 패0)
}

interface PlayerStats {
  appearances: number;
  goals: number;
  assists: number;
  momCount: number;
}
```

## 12. UI/UX Principles

### 디자인 원칙

1. **즉시 파악** — 구장 예약 가능 여부를 색상으로 즉시 구분 (초록/빨강)
2. **모바일 우선** — 이동 중에도 구장 검색·매칭 요청이 가능한 터치 친화적 UI
3. **최소 마찰** — 매칭 요청까지 3탭 이내
4. **투명한 데이터** — 팀·선수 통계는 항상 최신 경기 반영 여부를 표시
5. **커뮤니티 신뢰** — 실명 팀 기반 활동으로 신뢰성 확보

## 14. Risks & Assumptions

### Assumptions (가정)

- 노원구 공설 구장의 예약 API 또는 데이터 수집 협조 가능
- 사설 구장은 파트너십 또는 수동 등록 방식으로 시작
- 아마추어 팀 매니저들이 앱을 통한 매칭 방식에 적응 의지 있음

### Risks (위험 요소)

| 위험 | 영향 | 대응 |
|------|------|------|
| **구장 데이터 수집** — 공설 구장 API 미제공 | 높음 | 스크래핑 또는 수동 등록, 구청 협력 추진 |
| **콜드 스타트** — 초기 팀 수 부족으로 매칭 어려움 | 높음 | 오픈채팅방 홍보, 초기 팀 인센티브 제공 |
| **결과 미입력** — 경기 후 결과 등록 이탈 | 중간 | 자동 리마인더 알림, 입력 시 포인트 지급 |
| **정보 신뢰성** — 취소 없이 노쇼 발생 | 중간 | 노쇼 패널티 제도, 평판 점수 도입 |
| **구장 예약 중복** — 실시간 동기화 문제 | 중간 | Redis 기반 락(lock) 처리, 낙관적 잠금 |
